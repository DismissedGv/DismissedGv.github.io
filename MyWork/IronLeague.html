Iron League

About

Iron League is a mobile game inspired by a real-life intense sport. It was FutureGames second game project, developed in collaboration with Turborilla.

The game was built with C# and Unity together with a team of 17 people assigned by FutureGames.

Info

Role: Gameplay Programmer & Lead

Team Size: 17

Time: 4 Weeks

Engine: Unity

Introduction

In this project I took the responsibility of programming the game's audio system and user interface. Our team utilized Github desktop as source control, Miro for prototyping, Jira for project management. This was the first time I ever did a mobile game, but it wasn't so much different when it came down to logic.

Audio System

The game's audio system consists of three key scripts:

AudioManager: Responsible for managing volume levels and utilizing Unity's save and load system to persist volume settings.

Sound: A designer-friendly interface for working with the audio system, providing flexibility and ease of use.

UISoundController: Handles user input for volume adjustments in the game's settings menu, ensuring a seamless user experience.

User Interface

I have implemented two UI scripts in the game: ScoreManager and ScorePopup.

The ScoreManager script contains two key functions: one for initializing player scores and another for updating scores in real-time.

Meanwhile, the ScorePopup script is responsible for dynamically animating the score display using a lerp function to smoothly transition to the total score.

What I learned

With such a great team, I had the opportunity to refine my skills in general knowledge of C#, Unity for mobile development and project management tools like github, miro, and jira.
